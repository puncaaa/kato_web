# Техническое задание — улучшение сайта КАТО (страницы /about и /contacts)

Цель
Сделать современные, лёгкие, доступные и удобные страницы "О нас" (/about) и "Контакты" (/contacts) без потери существующего контента. RU по умолчанию, предусмотреть переключатель KZ/EN.

---

## 1. Общие правила оформления (глобально)
- Шрифт: Inter (weights 400, 600). Базовый размер — 16px; line-height: 1.5.
- Цвета: основной — #005BAC; фон страницы — #FFFFFF; блоки — #F7F9FC; текст — #072A3A; muted — #6c757d.
- Контейнер: max-width: 1280px; padding: 20px (mobile), 40px (desktop).
- Кнопки: radius 8px; тень 0 2 8 rgba(0,0,0,.08); hover — увеличенная тень. На моб. кнопки 100% ширина.
- Карточки: border-radius 10–12px; тень 0 4 16 rgba(0,0,0,.06).
- Анимация: reveal-on-scroll (AOS или lightweight IntersectionObserver).
- Иконки: Heroicons (outline) — единый стиль.
- Изображения: WebP, max 120 KB, обязательный alt.
- Доступность: WCAG 2.1 (контрастность), aria-label для иконок/кнопок.

---

## 2. Страница «О нас» (/about)
Что оставить
- Сохранить текст: миссия, стратегические направления, руководство.

Что убрать / изменить
- Разбить большие абзацы на карточки.
- Добавить оглавление‑якоря (миссия, направления, руководство, документы).

Структура страницы
1. Хедлайн: H1 + краткая миссия (1–2 предложения).
2. Оглавление‑якоря.
3. Миссия: CardIcon (иконка + текст).
4. Стратегические направления: 8 карточек CardIcon (иконка + заголовок + 1–2 строки). Сетка 4×2 desktop / 2×4 mobile.
   - Иконки однотонные, 48×48, consistent.
   - Карточки кликабельны (опционально открывают модал).
5. Руководство: список CardPerson — фото круглое 150×150 px, ФИО, должность, краткая биография.
6. Документы: список PDF (Устав и т.д.) — кнопка «Скачать».
7. CTA внизу: «Вступить в КАТО» (primary).

Визуал
- Заголовки: H1 28–32px; H2 22–24px.
- Отступы: между секциями 32–48px; карточки gap 20–32px.
- Все изображения lazy loading.

SEO/Meta
- Уникальный title/description/og:image для /about.

## 3. Страница «Контакты» (/contacts)
Что оставить
- Адрес, email, телефон, форму обратной связи.

Что убрать / исправить
- Убрать нерабочие внешние iframes; интегрировать корректный Google Maps embed (iframe) с маркером.

Структура
1. ContactRow блоки (иконка + текст + кнопка «Копировать» для телефона).
2. Google Maps iframe справа (height 360px). На мобильном — iframe скрыть и показать ссылку «Открыть в Google Maps».
3. FormContact (имя, email, сообщение, кнопка «Отправить»).
   - После отправки: показать «Спасибо, мы свяжемся».
   - Защита: reCAPTCHA v3 (серверная проверка).
   - Сохранение в ContactMessage + отправка email на trauma@nscto.kz.

UX
- Кнопка «Копировать» использует Clipboard API; fallback для старых браузеров.
- Форма валидируется на фронте и бэке.

---

## 4. Мобильная версия (везде)
- Минимальный размер текста 16px.
- Кнопки 48px высота, 100% ширина.
- Скрывать карту на моб. и показывать ссылку.
- Длинные тексты — "Показать ещё" (progressive disclosure).

---

## 5. Компоненты (переиспользуемые)
- CardIcon: icon, title, text, link? (для направлений).
- CardPerson: image, name, role, bio (руководство).
- ContactRow: icon, label, value, copyButton.
- FormContact: name, email, message, submit + recaptcha hook.
- SliderLogos: локальные логотипы партнёров (swiper/aria).

Расположение: templates/website/partials/{card_icon.html, card_person.html, contact_row.html, form_contact.html, slider_logos.html}

## 6. Техническое
- Технологии: Django templates, HTML5, CSS3, JS (ES6). Минимизировать внешние зависимости (no jQuery if possible).
- Статические ресурсы: webkato/static/website/{css,js,img,icons}.
- Каждый шаблон, где используется {% static %} — содержит {% load static %}.
- Контакты: views.contacts сохраняет ContactMessage и отправляет письмо.
- Файлы: все изображения хранить локально (WebP).
- reCAPTCHA v3: настроить ключи в settings, серверная проверка перед сохранением формы.
- SEO: meta и OG для /about и /contacts.
- Accessibility: aria, keyboard navigation, form labels.

## 7. Критерии приёмки
- Вёрстка соответствует правилам (шрифт, цвета, кнопки).
- /about и /contacts валидно отображаются на 375/768/1440.

- Код читаемый, шаблоны модульные (partials).
